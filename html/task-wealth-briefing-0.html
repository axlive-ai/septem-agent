모닝브리핑 · 사용자 케이스 정의

국내외 뉴스·자산 정보를 연결해 개인화 리포트를 제공하는 Sub Agent가 마주하는 사용자 케이스를 정의해요.

Case A · 첫 브리핑 수신

정의
처음 브리핑을 받는 고객이에요.
인식 신호
리포트 발송 이력 없음
웰컴 화면 방향
브리핑 구성(자산현황·이슈·영향분석·일정)을 간단히 소개해요.

Case B · 며칠간 미확인

정의
여러 날 브리핑을 확인하지 않은 고객이에요.
인식 신호
3일 이상 미열람 누적
웰컴 화면 방향
누적 기간의 핵심 변동사항만 요약해 보여줘요.

Case C · 특정 이슈 후속 질문

정의
브리핑 내용에 대해 추가로 질문하는 고객이에요.
인식 신호
브리핑 확인 후 관련 질문 발화
웰컴 화면 방향
해당 이슈에 대한 상세 분석을 추가로 제공해요.

Case D · 비대면 요약 선호

정의
WM 대면 상담 없이 핵심만 보는 고객이에요.
인식 신호
비대면 채널(올원뱅크)만 이용
웰컴 화면 방향
Weekly Report 전체 대신 1~2줄 핵심 요약을 제공해요.

케이스별 변형 포인트

화면 설계 시 고려 사항
Case A는 슬롯 0개 상태에서 시작 — 풀 플로우 필요
Case B는 중간 상태에서 시작 — 이어하기 카드 UI 별도 필요
Case C는 최근 이력 참조 UI 필요
Case D는 맥락이 사전에 채워진 상태로 시작 — 프리필 로직 필요
