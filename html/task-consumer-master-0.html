AI소비자보호 Master · 사용자 케이스 정의

소비자보호 AI가 마주하는 사용자 케이스를 정의해요. 긴급 상황이 많아 일반 케이스와 다르게 접근해요.

Case A · 직접 진입

정의
스스로 피해를 의심해 직접 진입한 고객이에요.
인식 신호
메뉴 클릭으로 진입 · 긴급 알림 없음
웰컴 화면 방향
차분하게 "무엇을 도와드릴까요?"로 시작하며 4가지 서비스를 안내해요.

Case B · 실시간 알림 진입

정의
피싱 탐지 알림을 받고 즉시 진입한 고객이에요.
인식 신호
SDK 탐지 알림 클릭 · 위험 정보 사전 확보
웰컴 화면 방향
알림 내용을 즉시 보여주고 대응 절차로 직행해요.

Case C · 고위험 발화 즉시 감지

정의
첫 발화에서 이미 고위험 상황이 드러나는 고객이에요.
인식 신호
"계좌번호 알려줬어요" 등 고위험 키워드 포함
웰컴 화면 방향
확인 절차 최소화하고 즉시 지급정지 유도로 전환해요.

Case D · 예방 정보 탐색

정의
피해는 없지만 예방 정보를 찾는 고객이에요.
인식 신호
긴급 키워드 없음 · "예방" "주의" 등 정보성 발화
웰컴 화면 방향
긴급 모드 대신 정보 안내 모드로 차분하게 전환해요.
