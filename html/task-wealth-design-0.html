포트폴리오 설계 · 사용자 케이스 정의

진단 결과에 따른 조정안을 제안하는 Sub Agent가 마주하는 사용자 케이스를 정의해요.

Case A · 진단 직후 설계 요청

정의
진단 결과 확인 직후 조정 설계를 요청한 고객이에요.
인식 신호
직전 단계가 포트폴리오 진단 완료
웰컴 화면 방향
진단 결과를 그대로 이어받아 조정 방향을 제안해요.

Case B · 제안 거부 후 보류

정의
제안을 거부하고 나중으로 미룬 고객이에요.
인식 신호
"나중에" 선택 이력 존재
웰컴 화면 방향
강요하지 않고 리마인더 설정 여부를 묻는 정도로 마무리해요.

Case C · 부분 실행

정의
제안 중 일부만 실행한 고객이에요.
인식 신호
이전 제안 일부 실행 완료 데이터 존재
웰컴 화면 방향
남은 조정 사항을 다시 안내하며 진행 여부를 물어요.

Case D · 위험 성향 변경

정의
이전과 다른 위험 성향을 보이는 고객이에요.
인식 신호
위험 성향 재설문 결과 변경 감지
웰컴 화면 방향
변경된 성향을 반영해 새로운 조정 상한을 다시 산출해요.
