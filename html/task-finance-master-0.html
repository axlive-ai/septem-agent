금융비서 Master · 사용자 케이스 정의

AI 금융비서가 마주하는 사용자 케이스를 정의해요.

Case A · 첫 방문

정의
AI 금융비서를 처음 이용하는 고객이에요.
인식 신호
AI 금융비서 이용 이력 없음
웰컴 화면 방향
서비스 유형(상품·계산) 소개 후 선택을 유도해요.

Case B · 의도 불명확

정의
발화에서 명확한 의도가 파악되지 않는 고객이에요.
인식 신호
추상적인 발화 ("뭐 좀 알려줘" 등)
웰컴 화면 방향
4가지 대분류 Chip을 제시해 선택을 유도해요.

Case C · 재방문

정의
이전에 상품 추천이나 계산을 이용한 고객이에요.
인식 신호
7일 이내 이용 이력 존재
웰컴 화면 방향
최근 이용 내용을 참조해 "오늘도 같은 도움 드릴까요?"로 물어요.

Case D · Sub Agent 간 전환

정의
한 Sub Agent 이용 중 다른 의도가 감지된 고객이에요.
인식 신호
상품 추천 중 계산 키워드 감지 등
웰컴 화면 방향
"잠깐 계산기로 전환할게요!" 식으로 끊김 없이 전환해요.

케이스별 변형 포인트

화면 설계 시 고려 사항
Case A는 슬롯 0개 상태에서 시작 — 풀 플로우 필요
Case B는 중간 상태에서 시작 — 이어하기 카드 UI 별도 필요
Case C는 최근 이력 참조 UI 필요
Case D는 맥락이 사전에 채워진 상태로 시작 — 프리필 로직 필요
