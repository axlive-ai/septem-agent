웰컴대화 · 사용자 케이스 정의

전체 Agent 공통으로 적용되는 웰컴 케이스를 정의해요. 모든 Agent의 사용자 케이스 정의서가 이 문서를 기준으로 구체화돼요.

Case A · 첫 방문

정의
AI Agent를 처음 이용하는 고객이에요. 서비스 경험이 없어요.
인식 신호
AI 이용 이력 없음 · 온보딩 미완료 상태 · 첫 GNB AI 버튼 클릭
웰컴 화면 방향
AI 소개 + 서비스 안내 중심. 컨텍스트 카드 없이 "무엇을 도와드릴까요?"로 시작해요.

Case B · 진행 중 이탈

정의
대화나 슬롯 입력 중 화면을 벗어났거나 앱을 종료한 고객이에요.
인식 신호
24시간 이내 미완료 세션 존재 · 일부 슬롯만 입력된 상태
웰컴 화면 방향
이어하기 카드로 중단 시점 상태를 보여주고, "이어서 진행할까요?" 칩을 제공해요.

Case C · 완료 후 재방문

정의
이전에 서비스를 완료한 경험이 있는 고객이에요.
인식 신호
7일 이내 동일 Agent 이용 이력 · 최근 결과 데이터 존재
웰컴 화면 방향
"지난번 [이용내용] 도와드렸었죠" 형태로 최근 이력을 참조해 인사해요.

Case D · 특정 경로 진입

정의
알림, 추천, 다른 Agent에서 특정 맥락으로 자동 전환된 고객이에요.
인식 신호
푸시 알림 클릭 · 다른 Agent에서 의도 감지 후 전환
웰컴 화면 방향
맥락을 그대로 이어받아 "알려드린 내용 확인하러 오셨군요!" 식으로 즉시 반응해요.

Case A는 슬롯 0개 상태에서 시작 — 풀 플로우 필요
Case B는 중간 상태에서 시작 — 이어하기 카드 UI 별도 필요
Case C는 최근 이력 참조 UI 필요
Case D는 맥락이 사전에 채워진 상태로 시작 — 프리필 로직 필요
