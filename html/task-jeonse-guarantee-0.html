보증추천 · 사용자 케이스 정의

보증기관 비교·추천 Sub Agent가 마주하는 사용자 케이스를 정의해요.

Case A · 첫 보증 조회

정의
처음 보증 한도를 조회하는 고객이에요.
인식 신호
준비 단계 최초 진입 · 보증 조회 이력 없음
웰컴 화면 방향
기본 정보(자금·연령 등) 수집부터 친절하게 시작해요.

Case B · 정보 입력 중 이탈

정의
정보 입력 중 이탈한 고객이에요.
인식 신호
일부 정보만 입력된 미완료 세션
웰컴 화면 방향
이어하기로 중단된 입력부터 재개해요.

Case C · 조건 변경 재조회

정의
조건을 바꿔 다시 조회하는 고객이에요.
인식 신호
이전 조회 결과 존재 · 조건 변경 요청
웰컴 화면 방향
이전 결과를 참조해 변경된 조건만 빠르게 재수집해요.

Case D · 청년·신혼 우대 대상

정의
청년·신혼부부 우대 조건에 해당하는 고객이에요.
인식 신호
연령·가구구성 정보에서 우대 조건 매칭
웰컴 화면 방향
우대 혜택을 강조해 안내하며 자긍심을 느끼게 해요.

케이스별 변형 포인트

화면 설계 시 고려 사항
Case A는 슬롯 0개 상태에서 시작 — 풀 플로우 필요
Case B는 중간 상태에서 시작 — 이어하기 카드 UI 별도 필요
Case C는 최근 이력 참조 UI 필요
Case D는 맥락이 사전에 채워진 상태로 시작 — 프리필 로직 필요
