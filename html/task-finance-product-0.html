맞춤상품안내 · 사용자 케이스 정의

맞춤 상품 안내 Sub Agent가 마주하는 사용자 케이스를 정의해요.

Case A · 첫 추천

정의
처음 상품 추천을 받는 고객이에요.
인식 신호
AI 금융비서에서 최초 연결 · 추천 이력 없음
웰컴 화면 방향
친절한 니즈 수집부터 시작해요 — "목표 기간이 어떻게 되세요?"

Case B · 슬롯 수집 중 이탈

정의
니즈 슬롯(기간·금액 등) 입력 중 이탈한 고객이에요.
인식 신호
미완료 슬롯 세션 존재
웰컴 화면 방향
이어하기 카드로 중단된 슬롯부터 재개해요.

Case C · 이미 가입 고객

정의
추천 대상 상품에 이미 가입한 고객이에요.
인식 신호
가입 상품 데이터와 추천 후보 중복
웰컴 화면 방향
다음 우선순위 상품으로 자연스럽게 전환해요.

Case D · 만기·금리 알림 진입

정의
만기 알림이나 금리 변동 알림으로 진입한 고객이에요.
인식 신호
알림 클릭 진입 · 알림 관련 상품 ID 존재
웰컴 화면 방향
알림 맥락을 그대로 이어받아 관련 상품을 즉시 안내해요.

케이스별 변형 포인트

화면 설계 시 고려 사항
Case A는 슬롯 0개 상태에서 시작 — 풀 플로우 필요
Case B는 중간 상태에서 시작 — 이어하기 카드 UI 별도 필요
Case C는 최근 이력 참조 UI 필요
Case D는 맥락이 사전에 채워진 상태로 시작 — 프리필 로직 필요
