방법론

NH AI Agent UX Guide System은 Agent Profile → Agent Task Design → Agent UX Design → Prototype 순서로 구성돼요. 각 단계가 어떤 역할을 하고, 서로 어떻게 연결되는지 설명해드릴게요.

4단계로 완성하는 설계 방식

4단계 구성
STEP 01 · Agent Profile — 이 AI가 누구인지 정의해요.
STEP 02 · Agent Task Design — 어떻게 일하는지 설계해요.
STEP 03 · Agent UX Design — 어떻게 보이는지 정의해요.
STEP 04 · Prototype — 실제로 작동하게 만들어요.

단계가 연결되는 방식

HOW IT CONNECTS
Profile에서 정의한 역할이 Task Design의 업무 흐름을 만들고, 그 흐름이 UX Design의 화면 기준을 결정하며, Prototype에서 실제 경험으로 구현돼요.

각 단계 상세

각 단계에서 무엇을 정의하고, 어떤 산출물이 나오는지 정리했어요.

1 · Agent Profile — 이 AI 에이전트가 누구인지 한 페이지로 정의
각 에이전트의 이름, 역할, 수행 범위, 다른 에이전트와의 관계를 명확히 정의해요. 설계를 시작하기 전에 "이 에이전트는 어떤 고객의 어떤 문제를 해결하는가"를 먼저 정의하는 단계예요. Profile이 명확해야 Task Design이 올바른 방향으로 진행돼요.
🪪 Agent Profile 문서 — 에이전트 역할 정의 · Master/Sub 관계 · 담당 업무 범위 · 진입 경로
2 · Agent Task Design — 고객과 어떻게 대화하고, 업무를 어떻게 처리하는지 정의
에이전트가 고객과 실제로 나누는 대화 흐름과 업무 처리 방식을 상세히 정의해요. 화면(스크린)이 아닌 업무(Task)를 기준으로 설계해요. 같은 화면 안에서도 대화가 진행되면서 상태가 계속 바뀌기 때문에, 업무 단위로 정의하는 것이 AI 에이전트 설계의 핵심이에요.
💬 대화흐름정의 — 고객 케이스 정의(Case A~D) · 케이스별 실제 대화 · 발화 정의 · UI 컴포넌트 매핑
📋 업무 설계서 — 업무 단계별 조건 · UI 표현 · 예외 처리 · 실제 화면 상태(mockup)
3 · Agent UX Design — 모든 에이전트가 공통으로 따르는 디자인 기준 정의
각 에이전트가 개별적으로 다른 디자인을 쓰면 고객 경험이 일관되지 않아요. 색상, 타이포그래피, 컴포넌트, 레이아웃, 인터랙션 원칙을 공통 기준으로 정의해서 어떤 에이전트든 같은 경험을 제공하도록 해요.
🎨 Foundation — 색상 · 타이포그래피 · 그리드 · 간격 시스템
📐 Layout · Interaction · Component — 화면 구성 원칙 · 대화 인터랙션 패턴 · 재사용 가능한 UI 컴포넌트
4 · Prototype — 실제로 눌러보면서 경험을 검증하는 단계
앞 단계에서 정의한 대화 흐름, 업무 설계, UX 기준을 바탕으로 실제로 작동하는 인터랙티브 프로토타입을 만들어요. 개발이 시작되기 전에 실제 경험을 먼저 확인하고, 이해관계자 리뷰와 사용자 테스트에 활용해요.
▶️ 인터랙티브 프로토타입 — 실제 대화 흐름을 체험할 수 있는 클릭 가능한 프로토타입
