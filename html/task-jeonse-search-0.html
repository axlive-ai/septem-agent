매물탐색 · 사용자 케이스 정의

예산·시세 기반 매물 탐색 Sub Agent가 마주하는 사용자 케이스를 정의해요.

Case A · 보증 추천 직후 진입

정의
보증 한도 확인 직후 자연스럽게 매물 탐색으로 넘어온 고객이에요.
인식 신호
직전 단계가 보증추천 완료
웰컴 화면 방향
확인된 예산 한도를 그대로 적용해 매물을 안내해요.

Case B · 매물탐색 메뉴 직접 진입

정의
보증 추천 없이 매물탐색 메뉴로 바로 들어온 고객이에요.
인식 신호
예산 한도 데이터 없음
웰컴 화면 방향
예산 입력을 먼저 요청하거나 보증추천을 권장해요.

Case C · 제휴 페이지 복귀

정의
제휴사 페이지에서 매물을 보고 돌아온 고객이에요.
인식 신호
제휴 페이지 방문 이력 존재
웰컴 화면 방향
다른 매물도 보고 싶은지 묻고 다음 단계를 안내해요.

Case D · 예산 변경 재탐색

정의
예산을 변경해 다시 탐색하는 고객이에요.
인식 신호
예산 한도 변경 요청
웰컴 화면 방향
변경된 예산으로 즉시 다시 매물 상세보기를 안내해요.
