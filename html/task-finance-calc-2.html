AI금융계산기 · 업무 설계서

Agent가 어떤 업무를 어떻게 수행하는지 정의해요. 업무 정의, 업무 흐름, 단계별 상세, 엣지 케이스를 다뤄요.

업무 정의

업무명
금융 계산 — 고객이 원하는 금융 수치를 대화를 통해 계산하고 결과를 제공해요.
업무 유형
룰형 (슬롯 필링) — 필요한 정보를 순차적으로 수집하여 정해진 계산식으로 결과를 도출해요.
수행 주체
Agent(계산 실행·결과 제공) + 고객(계산 조건 입력·선택)
업무 범위
대출이자 · 전세이자 · 적금만기 · DSR 계산 (본 문서는 대출이자 기준)
완료 조건
계산 결과를 고객이 확인하고 다음 행동(재계산·상품추천·종료)을 선택해요.

업무 흐름

시작
업무 1계산 유형 파악
2-1 대출금액
2-2 금리
2-3 상환기간
2-4 상환방식
업무 3계산·결과 제공
분기 처리
재계산 → 변경 슬롯만 교체 후 업무 3 복귀
상품추천 → 맞춤 상품 안내 Agent로 전환
범위 초과·주제 이탈 → 오류 안내 후 동일 슬롯 재수집

실제 서비스 엣지 케이스

개발 협의 필요
세션 중단 — 슬롯 수집 중 앱 종료 시 상태 저장 유효 기간 (1일? 7일?)
네트워크 오류 — 응답 타임아웃 기준 및 재시도 최대 횟수, 3회 실패 시 인간 채널 연결 여부
로그인 세션 만료 — 슬롯 상태 임시 저장 방식, 비로그인 상태에서 계산 허용 여부
계산 중 다른 업무 요청 — 다른 업무 완료 후 복귀 여부, 슬롯 임시 보관 유효 기간
