금융사기 예방 · 사용자 케이스 정의

고객 프로파일 기반 맞춤형 예방 안내 Sub Agent가 마주하는 사용자 케이스를 정의해요.

Case A · 취약 고객군 첫 노출

정의
취약 사기 유형 매핑이 처음 적용되는 고객이에요.
인식 신호
고령층·지방 거주 등 취약 프로파일 신규 매핑
웰컴 화면 방향
해당 고객군의 대표 사기 유형을 친절하게 소개해요.

Case B · 예방 정보 무시 후 재노출

정의
이전 예방 정보를 확인하지 않은 고객이에요.
인식 신호
이전 노출 이력 존재 · 미확인 상태
웰컴 화면 방향
다른 표현·사례로 재구성해 다시 노출해요.

Case C · 예방 정보 확인 후 재방문

정의
이전에 예방 정보를 확인한 고객이에요.
인식 신호
확인 완료 이력 존재
웰컴 화면 방향
새로운 사기 유형이나 추가 정보를 우선 제공해요.

Case D · 실제 피해 위험 신호 감지

정의
안내 중 실제 위험 신호가 감지된 고객이에요.
인식 신호
예방 안내 중 피해 의심 발화 감지
웰컴 화면 방향
즉시 피싱탐지 또는 소비자보호 Master로 긴급 전환해요.
