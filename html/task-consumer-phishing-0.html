피싱탐지 · 사용자 케이스 정의

SDK 기반 실시간 피싱 탐지 Sub Agent가 마주하는 사용자 케이스를 정의해요. 시스템 자동 트리거형이에요.

Case A · 악성 URL 클릭 시도

정의
스미싱 문자의 악성 URL을 클릭하려는 순간이에요.
인식 신호
SDK가 악성 URL 패턴 실시간 탐지
웰컴 화면 방향
전면 팝업으로 즉시 경고하고 클릭을 차단해요.

Case B · 악성 앱 실행 시도

정의
원격제어 등 악성 앱을 실행하려는 순간이에요.
인식 신호
SDK가 악성코드 실행 시도 탐지
웰컴 화면 방향
앱 실행을 차단하고 위험성을 즉시 안내해요.

Case C · 백그라운드 상태 탐지

정의
앱이 백그라운드 상태일 때 위험이 감지돼요.
인식 신호
백그라운드 상태에서 SDK 탐지 이벤트 발생
웰컴 화면 방향
Push/Overlay로 즉각적인 경고창을 띄워요.

Case D · 반복 노출 고객

정의
이미 여러 차례 경고를 받은 고객이에요.
인식 신호
동일 세션 내 경고 2회 이상 발생
웰컴 화면 방향
단순 반복이 아닌 패턴 설명을 추가해 경각심을 높여요.

Case B는 중간 상태에서 시작 — 이어하기 카드 UI 별도 필요
Case C는 최근 이력 참조 UI 필요
Case D는 맥락이 사전에 채워진 상태로 시작 — 프리필 로직 필요
