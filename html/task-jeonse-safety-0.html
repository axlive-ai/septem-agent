깡통전세방지 · 사용자 케이스 정의

등기부·전세가율 분석 위험 진단 Sub Agent가 마주하는 사용자 케이스를 정의해요.

Case A · 매물탐색 직후 진단

정의
매물을 확인한 직후 위험 진단으로 넘어온 고객이에요.
인식 신호
직전 단계가 매물탐색 완료, 매물 주소 데이터 존재
웰컴 화면 방향
확인된 매물 주소를 그대로 적용해 진단을 시작해요.

Case B · 직접 진단 요청

정의
특정 매물 없이 진단 메뉴로 바로 들어온 고객이에요.
인식 신호
매물 주소 데이터 없음
웰컴 화면 방향
전세금·주소 입력을 먼저 요청해요.

Case C · 위험 판정 후 재확인

정의
위험 판정을 받은 후 다시 확인하는 고객이에요.
인식 신호
이전 진단 결과 존재 · 위험 등급 주의 이상
웰컴 화면 방향
이전 판정을 다시 보여주고 대안(전문가 상담 등)을 강조해요.

Case D · 안전 판정 후 계약 진행

정의
안전 판정을 받고 계약을 진행하려는 고객이에요.
인식 신호
이전 진단 결과 안전 등급
웰컴 화면 방향
안전 확인을 다시 안내하고 사후관리 단계를 제안해요.
