AI금융계산기 · 사용자 케이스 정의

공통 케이스 4가지를 AI금융계산기 맥락에 맞게 구체화해요. 케이스에 따라 웰컴 화면과 진입 흐름이 달라져요.

Case A · 첫 방문

정의
AI 금융계산기를 처음 이용하는 고객이에요. 계산 유형·진행 방식에 대한 경험이 없어요.
인식 신호
계산기 이용 이력 없음 · AI 금융비서에서 최초 연결 · 첫 슬롯 필링 진입
웰컴 화면 방향
계산 유형 선택부터 친절하게 안내해요. "어떤 계산을 도와드릴까요?" + 4가지 유형 Chip을 제시해요.

Case B · 진행 중 이탈

정의
슬롯 입력 중 화면을 벗어났거나 앱을 종료한 고객이에요. 계산이 미완료 상태예요.
인식 신호
24시간 이내 미완료 슬롯 세션 존재 · 일부 슬롯만 입력된 상태
웰컴 화면 방향
이어하기 카드로 중단 시점의 슬롯 상태를 보여주고, "이어서 계산할까요?" 칩을 제공해요.

Case C · 완료 후 재방문

정의
이전에 계산을 완료한 경험이 있는 고객이에요. 같은 계산을 반복하거나 다른 유형을 시도해요.
인식 신호
7일 이내 동일 Agent 이용 이력 · 최근 계산 결과 저장 데이터 존재
웰컴 화면 방향
"지난번 대출이자 계산 도와드렸었죠. 오늘도 같은 계산 도와드릴까요?" 형태로 최근 이력을 참조해요.

Case D · 특정 경로 진입

정의
상품 추천·알림 등 특정 맥락에서 계산 의도가 감지되어 자동 전환된 고객이에요.
인식 신호
AI 금융비서 대화 중 계산 키워드 감지 · 상품 상세 화면에서 "이자 계산" 버튼 클릭
웰컴 화면 방향
맥락을 그대로 이어받아 "잠깐 계산기로 전환할게요!" 후 관련 슬롯(금리 등)을 미리 채워서 시작해요.

케이스별 변형 포인트

화면 설계 시 고려 사항
Case A·D는 슬롯 0개 상태에서 시작 — 풀 플로우 필요
Case B는 중간 상태에서 시작 — 이어하기 카드 UI 별도 필요
Case C는 최근 결과 참조 UI 필요 — 저장된 계산 결과 데이터 연동
Case D는 일부 슬롯이 사전에 채워진 상태로 시작 — 슬롯 프리필 로직 필요
